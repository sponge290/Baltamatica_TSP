# 天气感知型 TSP 求解系统 内网部署指南

## 一、部署环境准备

### 1.1 硬件要求
- 内网服务器：2核4G以上，硬盘50G以上
- 支持Windows Server或Linux系统
- 可连通内网用户终端

### 1.2 软件要求
- PostgreSQL 15+
- Python 3.8+
- Nginx 1.20+（推荐）
- 北太天元v3.6及以上（用户本地安装）

### 1.3 网络要求
- 内网服务器需开放以下端口：
  - 8080（前端访问）
  - 8000（API服务）
  - 5432（数据库，仅对内网API服务开放）

## 二、部署步骤

### 2.1 数据库部署

1. **安装PostgreSQL**
   - Windows：下载并安装PostgreSQL 15+，启用PostGIS扩展
   - Linux：使用包管理器安装：`apt install postgresql postgresql-contrib postgis`

2. **初始化数据库**
   ```bash
   # 进入PostgreSQL命令行
   psql -U postgres
   
   # 创建数据库
   CREATE DATABASE tsp_db;
   
   # 退出
   \q
   ```

3. **执行初始化脚本**
   ```bash
   psql -U postgres -d tsp_db -f project/scripts/db_init.sql
   ```

4. **验证数据库连接**
   ```bash
   psql -U tsp_user -d tsp_db
   \dt  # 查看表结构
   ```

### 2.2 API服务部署

1. **安装依赖**
   ```bash
   cd project/api
   pip install -r requirements.txt
   ```

2. **配置数据库连接**
   - 修改 `main.py` 中的数据库连接参数（如果需要）

3. **启动API服务**
   ```bash
   # 开发模式
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   
   # 生产模式（推荐）
   uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
   ```

4. **验证API服务**
   访问 `http://服务器IP:8000/api/cities` 应返回城市数据

### 2.3 前端部署

1. **构建前端项目**
   ```bash
   cd project/frontend
   npm install
   npm run build
   ```

2. **配置Nginx**
   - 创建Nginx配置文件：`nginx.conf`
   ```nginx
   server {
       listen 8080;
       server_name localhost;
       
       # 前端静态文件
       location / {
           root project/frontend/dist;
           index index.html;
           try_files $uri $uri/ /index.html;
       }
       
       # API反向代理
       location /api {
           proxy_pass http://localhost:8000/api;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       }
       
       # 代理脚本下载
       location /proxy {
           alias project/proxy;
       }
   }
   ```

3. **启动Nginx**
   ```bash
   # Windows
   nginx -s reload
   
   # Linux
   systemctl restart nginx
   ```

4. **验证前端访问**
   访问 `http://服务器IP:8080` 应显示系统首页

## 三、服务管理脚本

### 3.1 API服务启动脚本
```bash
#!/bin/bash
# api_start.sh
cd project/api
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4 --log-file api.log
```

### 3.2 前端服务启动脚本
```bash
#!/bin/bash
# frontend_start.sh
cd project/frontend
npm run preview -- --port 8080
```

### 3.3 完整启动脚本
```bash
#!/bin/bash
# start_all.sh

# 启动API服务
echo "启动API服务..."
nohup bash api_start.sh > api_start.log 2>&1 &

# 启动前端服务
echo "启动前端服务..."
nohup bash frontend_start.sh > frontend_start.log 2>&1 &

echo "所有服务已启动"
```

## 四、本地代理配置

### 4.1 代理脚本配置
- 打开 `project/proxy/baltamatica_tsp_proxy.m`
- 修改CORS配置，添加内网前端地址：
  ```matlab
  config.allow_origin = {"http://localhost:3000", "http://服务器IP:8080"};
  ```

### 4.2 运行代理服务
1. 在北太天元v3.6中打开代理脚本
2. 点击运行按钮
3. 看到"天气感知型TSP 本地代理服务启动成功"提示

## 五、使用流程

1. **用户访问**：打开浏览器访问 `http://服务器IP:8080`
2. **连接代理**：点击"检测连接"按钮，确保本地代理服务已运行
3. **选择用例**：选择预设测试用例或自定义参数
4. **执行算法**：选择算法并点击"开始计算"
5. **查看结果**：查看可视化结果和性能指标
6. **保存分享**：保存结果并生成内网分享链接

## 六、常见问题排查

### 6.1 前端无法访问
- 检查Nginx服务是否运行
- 检查防火墙是否放行8080端口
- 检查服务器网络是否连通

### 6.2 API请求失败
- 检查API服务是否运行
- 检查数据库连接是否正常
- 检查Nginx反向代理配置

### 6.3 本地代理连接失败
- 检查北太天元是否运行
- 检查代理脚本是否执行
- 检查CORS配置是否正确
- 检查18080端口是否被占用

### 6.4 算法执行失败
- 检查北太天元版本是否为v3.6+
- 检查测试用例规模是否在算法支持范围内
- 查看前端日志中的错误信息

## 七、运维建议

### 7.1 定期备份
- 定期备份数据库：`pg_dump -U tsp_user tsp_db > tsp_db_backup.sql`

### 7.2 监控管理
- 监控API服务和前端服务状态

- 限制数据库访问权限
